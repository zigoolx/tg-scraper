آموزش مرحله به مرحله و خیلی ساده SNI Spoofing

۱. دانلود برنامه Patterniha

آخرین نسخه برنامه رو دانلود کنین (فایل zip یا rar).

https://github.com/patterniha/SNI-Spoofing/releases/tag/v1.0

۲. استخراج فایل‌ها

فایل رو استخراج کنین. داخل پوشه باید فایل Patterniha.exe باشه.

۳. فایل JSON رو آماده کنین

تو همون پوشه‌ای که Patterniha.exe هست، فایل config.json رو ویرایش کنین
بعد این محتوا رو دقیق کپی کنین داخلش:

{
  "LISTEN_HOST": "0.0.0.0",
  "LISTEN_PORT": 40443,
  "CONNECT_IP": "104.19.229.21",
  "CONNECT_PORT": 443,
  "FAKE_SNI": "www.hcaptcha.com",
  "QUEUE_NUM": 1000,
  "HANDSHAKE_TIMEOUT_MS": 2000
}

۴. برنامه رو اجرا کنین

- روی فایل Patterniha.exe راست کلیک کنین

- گزینه Run as administrator رو بزنین (خیلی مهمه!)

۵. کانفیگ V2Ray/Xray/Nekobox/v2rayNG رو تنظیم کنین

تو کانفیگی که داری فقط این دو تا رو عوض کن:

- Address یا آدرس سرور → 127.0.0.1
- Port یا پورت → 40443

بقیه چیزا (UUID، path، TLS، Reality و ...) رو همون قبلی نگه دار.

۶. وصل شو

کانفیگ رو ذخیره کن و بزن Connect. باید وصل بشه.

نکات مهم:

اگه پترنیها ران نشد یه بار از کانفیگ real delay بگیرن استارت میشه
- همیشه Patterniha رو به عنوان مدیر (Run as administrator) اجرا کن.
- اگه ویندوز فایروال پرسید، اجازه دسترسی بده.
- اگه وصل نشد: برنامه رو Stop کن، دوباره Start کن، یا یه config.json جدید با IP دیگه امتحان کن.

🚀 آپدیت Range Scout نسخه 0.7.0

این نسخه اسکن DNS و مرحله DNSTT را خیلی منعطف‌تر کرده. حالا هم در اسکن می‌تونی بین UDP / TCP / BOTH انتخاب کنی، هم در مرحله DNSTT از UDP / TCP / DOT / DOH استفاده کنی. تنظیمات جدید هم داخل UI و فایل config کامل ذخیره و لود می‌شن.

——————————————

✨ چی عوض شده:

🔹 پروتکل اسکن DNS قابل انتخاب شد
حالا اسکنر فقط UDP نیست و می‌تونی UDP، TCP یا BOTH را برای probeهای سازگاری انتخاب کنی

🔹 مرحله DNSTT حرفه‌ای‌تر شد
برای تست DNSTT حالا UDP، TCP، DOT و DOH پشتیبانی می‌شن

🔹 Resolver Endpoint اضافه شد
برای DNSTT می‌تونی endpoint دلخواه بدی و از placeholderهای {ip} و {port} استفاده کنی

🔹 تنظیمات جدید داخل config ذخیره می‌شن
فیلدهای protocol، transport و resolverURL حالا موقع Save Config نگه داشته می‌شن و دوباره درست لود می‌شن

🔹 اسکن TCP و حالت BOTH به هسته اسکن اضافه شد
نتایج resolverها حالا می‌تونن بر اساس عملکرد واقعی روی UDP و TCP ثبت بشن

🔹 تست‌ها و پایداری بهتر شد
تست‌های scanner، DNSTT و embedded client گسترده‌تر شدن و رفتار transportهای جدید پوشش داده شد

🔹 رابط کاربری و نسخه برنامه آپدیت شد
فیلدهای جدید DNSTT وارد UI شدن و نسخه داخل هدر روی v0.7.0 قرار گرفت
