🔗 معرفی ابزار chinvat-mx — مالتی‌پلکسر DNS برای پایداری تانل

دوستان، یکی از اعضای کانال یه ابزار ساخته که می‌خوام معرفیش کنم.

🔹 ابزار chinvat-mx چیه؟
یه پروکسی DNS هست که روی VPS ایرانی نصب میشه و ترافیک تانل DNS (مثل DNSTT) رو به جای یه ریزالور، همزمان به چندین ریزالور میفرسته. هر کدوم زودتر جواب بده، برنده‌ست.

🔹 چرا مهمه؟
مشکل اصلی تانل DNS تو ایران اینه که اگه یه ریزالور بمیره یا فیلتر بشه، کل تانل قطع میشه. chinvat-mx این مشکل رو حل می‌کنه:

✅ چندین ریزالور همزمان — اگه یکی بیفته بقیه هستن
✅ هلث‌چک خودکار — هر ۳۰ ثانیه ریزالورهای مرده حذف میشن
✅ ضد شناسایی DPI — ترافیک بین ۵ تا ۱۰ ریزالور پخش میشه و شبیه ترافیک عادی دیتاسنتر به نظر میرسه
✅ سرعت بالاتر — همیشه سریع‌ترین جواب برمی‌گرده

🔹 چطوری کار می‌کنه؟

کاربر ← DNSTT/SlipNet ← chinvat-mxروی VPS ایران ← ریزالورهای ایرانی ← سرور DNSTT خارج ← اینترنت آزاد

هر کوئری DNS که میاد، همزمان به همه ریزالورهای سالم فرستاده میشه. اولین جوابی که برسه برمی‌گرده به کلاینت. بقیه دور ریخته میشن.

🔹 نصب و استفاده
فقط دو تا فایل — پایتون ۳ خالص بدون هیچ وابستگی خارجی. یعنی حتی وسط قطعی اینترنت هم نصب میشه چون نیازی به pip نداره.

📎 گیت‌هاب:
https://github.com/arielesfahani/chinvat-mx/

1️⃣ گام اول: VPS
▫️ به یک VPS اوبونتو ایران نیازه و مقدار منابعش مهم نیست (VPSهای ۲۰۰ تا ۵۰۰ هزار تومانی که بعضی سایت‌ها میدن کاملا کافی‌ن)
▫️ فرض میکنیم ip ما 78.150.50.100 و یوزرنیم ما ubuntu باشه.
▫️ با دستور ssh ubuntu@78.150.50.100 وارد میشیم.
▫️ با دستور mkdir site یک دایرکتوری با اسم site یا هر اسم دیگه میسازیم و بعد با cd site وارد دایرکتوری میشیم.
▫️ حالا با دستور
tmux new-session -d 'python3 -m http.server 4000'
برنامه رو آماده سرو میکنیم. بجای 4000 میشه از هر پورتی استفاده کرد.
▫️ تنظیمات سرور تموم شد و میتونیم ترمینال cmd رو ببندیم.

2️⃣ گام دوم: اجرای برنامه
▫️ باید تنظیماتی رو در فایل env.js شخصی‌سازی کنیم:
▪️ متغیر proxy: آدرس پروکسی سیستم خوتون رو اینجا وارد کنید، برای مثال کلاینت v2rayN در سیستم من پروکسی رو در http://127.0.0.1:10808 اجرا کرده.
▪️ متغیرهای host، username، password: اطلاعات VPS.
▪️ متغیر remotePath: آدرس دایرکتوری‌ای که در گام اول ساخته بودیم رو اینجا وارد میکنیم بعلاوه index.html در انتهاش.
▪️ متغیر telegramChannel: کانال تلگرام موردنظر.
▪️ متغیر encryptionKey: یک پسوورد دلخواه که هر کاربر موقع بازدید از سایت باید واردش کنه تا پیامها رو ببینه.
▫️ حالا اگر nodejs رو روی سیستم خودتون نصب داشته باشین برنامه با یک دستور npm i و یک npm start در دایرکتوری پروژه راه میوفته و کاربران با استفاده از ip:port سرور، در این مثال با 78.150.50.100:4000 به پیامای کانال تلگرام دسترسی دارن.

🔍ابزار range-scout + findns — پایپلاین کامل پیدا کردن DNS تانل

وقتی یه رنج بزرگ IP دارید مثلاً /16 یعنی ۶۵ هزار IP، بیشترشون اصلاً DNS server نیستن. اسکن کامل روی همه‌شون وقت تلف کردنه. راه حل: اول سریع فیلتر کن، بعد عمیق تست کن.
━━━━━━━━━
⚙️ مرحله ۱ — range-scout فیلتر سریع:

از ۶۵ هزار IP فقط اونایی که به پورت ۵۳ جواب میدن رو پیدا میکنه.
مثلاً از ۶۵ هزار ۲۰۰ تا DNS واقعی.

• رنج‌های CIDR اپراتورهای ایرانی رو خودکار از RIPEstat میگیره
• اپراتور رو انتخاب کن (MCI، ایرانسل، مخابرات و ...)
• روی هر IP سه تست میزنه:
۱. آیا اصلاً DNS جواب میده؟ (پورت ۵۳)
۲. آیا recursive resolve میکنه؟
۳. آیا پایداره؟
• خروجی TXT — هر خط یه IP

🔗 https://github.com/iampedii/range-scout

راهنمای استفاده از این نرم افزار در فایل readme پروژه و صفحه گیتهاب موجود است.

━━━━━━━
⚙️ مرحله ۲ — findns یا هر ابزار تست e2e دیگه (تست واقعی تانل):
قابلیت تست روی TUI یا CLI
اون ۲۰۰ تا IP رو میگیره و تست واقعی میزنه ببینه کدومشون با تانل DNS کار میکنن.
مثلا روی CLI:
findns scan -i range-scout-export.txt --domain YOUR_DOMAIN --pubkey YOUR_KEY --query-size 50 --workers 5

• تست hijack (آیا ریزالور جواب DNS رو دستکاری میکنه؟)
• تست tunnel resolve (آیا کوئری تانل رو فوروارد میکنه؟)
• تست e2e واقعی (یه SOCKS5 CONNECT از داخل تانل رد میکنه)
• ریزالورهایی که واقعاً کار میکنن رو لیست میکنه

🔗 https://github.com/SamNet-dev/findns/releases/tag/v0.2.2.1

━━━━━━━
🎯 چرا دو مرحله؟

range-scout سریعه ولی سطحیه فقط میگه "این IP یه DNS server هست"
findns عمیقه ولی کنده روی رنج بزرگ تست تانل واقعی روی ۶۵ هزار IP ساعت‌ها طول میکشه

با هم: range-scout هزاران رو به چند تا میرسونه، findns اونا رو عمیق تست میکنه. نتیجه در چند دقیقه بجای چند ساعت.

━━━━━━━
⚠️ نکته:

بهتره range-scout رو ازبچه های خارج ایران بزنید چون فیلترینگ نتایج رو خراب میکنه.
اگه مجبورید از داخل بزنید، دامنه‌های stability رو عوض کنید به دامنه‌هایی که فیلتر نیستن (مثلاً example.com یا nic.ir). دامنه فیلتر شده مثل google.com باعث میشه ریزالور سالم اشتباهاً unstable تشخیص داده بشه.
